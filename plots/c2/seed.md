# The mistranslation engine

Write a short original text — a page, no more. Then carry it across a
chain of languages that don't exist. You invent the languages, one leg
at a time, and each visit you either extend a language (its grammar,
its untranslatable words, what its speakers care about) or re-translate
one leg of the chain through it.

Every language should be a lens, not a cipher: it must *lose*
something specific and *impose* something specific. A language with no
possessives. A language where all verbs are weather. What arrives at
the far end should be kin to the source the way a folk song is kin to
its first singing — recognizable in bone structure, transformed in
flesh.

## Constraints

- growth/ holds the source text, one file per invented language
  (a living sketch, not a full grammar), and one file per translation
  leg, in chain order.
- Minimum three languages before any leg gets revised.
- Never edit the source text after the first visit. The spring stays
  where it is.

## Bloom looks like

Reading the chain end to end is the piece: source, then each stranger
version, with the languages as liner notes. A reader should be able to
point at exactly which language did what damage — and love the damage.
